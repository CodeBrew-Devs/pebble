Three logo variants for different contexts. The wordmark is primary. The symbol mark works as a favicon or app icon. The lockup combines both for marketing contexts.

Logo Rules
✓ Always use Fraunces Bold (700) for the wordmark
✓ Minimum size: 80px wide for wordmark
✓ "pebble" in sage-dark (#4d7465) on light backgrounds
✓ "pebble" in sage (#7c9e8f) on dark backgrounds
✓ Dot "." always in contrast color (dark or white)
✗ Never stretch, rotate, or recolor outside the rules
✗ Never use on busy backgrounds without a clear field
✗ Never use all-black or all-white for the full wordmark
06 — Interaction

Motion & Animation

Motion in Pebble is calm and purposeful — never flashy. Transitions reinforce spatial relationships and provide gentle feedback. All durations are under 400ms.

TokenDurationEasingUsage
motion-instant100mseaseHover color changes, chip selections
motion-fast150–200mseaseButton hover, border focus, icon states
motion-default300mseasePage transitions, modal open/close
motion-deliberate350mscubic-bezier(.34,1.56,.64,1)Progress bars, goal card fills (spring)
motion-slow600–800mscubic-bezier(.34,1.56,.64,1)Budget split bar on dashboard load
Key Animations
fadeUp — Page entrance: opacity 0→1 + translateY(12px→0) over 350ms. Used on every page transition.
slideUp — Bottom sheet modal: translateY(60px→0) + opacity over 300ms with spring easing.
Progress fill — Goal bars animate width from 0→target on mount, 800ms spring, stagger 100ms per card.
Hover lift — Primary/sage buttons: translateY(-1px) + shadow on hover, 200ms ease.
07 — Brand

Voice & Tone

Pebble speaks like a knowledgeable, encouraging friend — never a stern financial advisor. Keep it warm, specific, and forward-looking.

✓ Do
Use encouraging, forward-looking language
Reference the user's specific goals by name
Celebrate milestones, even small ones
Use simple, jargon-free money language
Frame debt payoff as progress, not deficit
Use the pebble metaphor where natural
✗ Don't
Use banking jargon or formal financial terms
Guilt or shame users about overspending
Use vague, generic taglines that could be anything
Show negative numbers without context
Overload with data — simplicity is the product
Be preachy about financial decisions
